# 📄 DageAlbum 技术白皮书 (Technical Whitepaper)
**版本：** v1.0
**主题：** 基于 Nostr 协议扩展的“零信任”端到端加密云相册架构

### 1. 摘要 (Abstract)
在数据隐私泄漏频发、中心化云盘随意审查用户数据的时代，DageAlbum 提出了一种全新的个人数据主权解决方案。本项目摒弃了传统的“中心化信任”模型，创新性地将 **Nostr 去中心化协议** 与 **WebCrypto 军工级端到端加密 (E2EE)** 相结合。通过将文件粉碎为无法破解的 AES-GCM 密文碎片进行存储，确保服务器端对用户数据达到绝对的“零认知（Zero-Knowledge）”。

### 2. 核心密码学架构 (Cryptographic Architecture)
DageAlbum 的安全性建立在严格的密码学闭环之上，密钥绝不离开客户端内存。
*   **本地安全金库：** 用户的底层私钥 (Nostr nsec) 不以明文落地。采用 `PBKDF2` (高达 60 万次迭代防彩虹表碰撞) 结合用户的本地安全密码，生成强密钥后使用 `AES-256-GCM` 加密存入 LocalStorage。
*   **双独立 IV 媒体流变异：** 上传的每一张原图及其缩略图，均会被分配完全独立的初始化向量 (Nonce/IV)，并在前端的 `WebWorker` 多线程中完成 `AES-GCM` 碎流加密，彻底杜绝了同密钥下的 IV 复用雪崩漏洞。
*   **非对称密钥交换分享 (NIP-44/04)：** 当用户将照片分享给通讯录好友时，系统利用 `Secp256k1` 椭圆曲线算法生成 Diffie-Hellman 共享密钥，将照片的 AES 解密钥匙安全地送达目标公钥，实现了无中心机构介入的安全授权。

### 3. Nostr 协议的定制化延展 (Protocol Extension)
DageAlbum 打破了 Nostr 仅用于社交文本的局限，将其抽象为去中心化的文件状态机：
*   **底层鉴权 (NIP-98 升级版)：** 放弃传统的 Token 机制。所有 HTTP 请求（上传/下载/删除）均采用 NIP-98 Schnorr 签名鉴权，并将分片参数注入签名 Tags，不仅免疫了跨域拦截，配合后端的恒定时间校验与内存防重放池，彻底封死防重放攻击 (Replay Attack)。
*   **元数据引擎 (Kind 30400 / 30401)：** 将图片的哈希、加密 IV、原始拍摄时间（EXIF 提取）、自定义标签封装为密文存入 Nostr Event。
*   **物理级权限粉碎 (Kind 5 拦截)：** 创新实现软硬联动。当用户发出 Kind 5 撤回分享或删除指令时，系统不仅销毁链上记录，同时后端将对底层物理硬盘上的密文碎片执行原子级抹除，绝不留痕。

### 4. 极致工程与高并发防御 (Engineering & Resilience)
*   **微秒级原子锁 (Atomic Locks)：** 后端存储引擎采用 SQLite WAL 模式，配合 `rowcount` 事务级拦截，彻底消灭了并发删除/上传引发的“配额双重释放”与网盘容量无限刷取漏洞。
*   **防 OOM 内存管理：** 后端处理 10GB 级大视频上传时，采用 1MB 游标流式计算哈希；前端批量下载打包 (JSZip) 采用 200MB 弹性分卷与锯齿状内存回收机制，完美防御内存溢出 (OOM)。
*   **防误触沉浸式挂机：** 独创的“锁屏不锁后台”机制。超时或手动锁屏时，仅拦截 UI 并销毁会话凭证，但底层的加解密 Worker 与网络 I/O 依然静默运行。配合浏览器级的 `beforeunload` 强杀拦截，保障超大文件传输不因误操作而腰斩。

# 🌟 特性说明与竞品对比 (Feature Comparison)

### 为什么选择 DageAlbum？
市面上的云相册（如 iCloud、Google Photos、百度网盘）本质上是“你把钥匙和日记本一起交给了门卫”。他们不仅能看你的照片，还能用你的照片训练 AI，甚至因为误判封禁你的账号。而自己搭建的 NAS（如群晖、Nextcloud）虽然数据在自己手里，但一旦内网被黑客攻破，硬盘里的明文照片依然会泄露。

**DageAlbum 重新定义了相册：它不仅是一个展示工具，更是一个抗审查、防窃取的数字保险箱。**

### 💡 独家特性亮点
1. **服务器“物理致盲”：** 服务器上没有 jpg，没有 mp4，只有一堆乱码的 `.bin` 文件。即使服务器管理员拔走硬盘、甚至黑客拖走数据库，没有你脑子里的本地密码，全宇宙的算力也无法还原出一张完整的缩略图。
2. **极客级分享 (Crypto-Share)：** 抛弃不安全的“提取码链接”。分享照片时，系统利用公私钥为好友单点派发加密钥匙。除了该好友，哪怕数据在传输中途被拦截也无法解密。
3. **“阅后即焚”式物理撤回：** 发现发错照片了？点击“撤回”。系统会立刻向服务器发送粉碎指令，不仅摧毁授权凭证，更会直接抹除对方收件箱对应的真实文件关联。
4. **全平台流畅体验：** 单文件 HTML 架构，秒开运行。支持极速拖拽上传、本地标签聚合、无缝分块断点机制、以及极具赛博朋克感的“流水线安全打包下载”UI。
